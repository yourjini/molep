## 4. 확정된 핵심 요건 (2026-04-17 내부회의)

1. **IP 국가 감지**: MaxMind GeoIP2 DB, 주 2회 자동 갱신 (Cloudflare CF-IPCountry 헤더 fallback)
2. **기본 언어**: **한국어 고정** (모든 국가 동일). 구글 번역 위젯으로 다국어 처리
3. **가입 플로우**: 국가 선택 + 쿠키 검증 (IP와 쿠키 불일치 시 재선택 유도)
4. **Auth 연동**: 프론트 쿠키의 국가 값을 Auth 서버에 전달 (Auth는 직접 국가 확인 불가)
5. **어드민**: 콘텐츠 노출 국가 필터링 (국가별 콘텐츠 분기)

### 국가별 인증 수단 & 연령 제한

| 국가 | 인증 수단 | 최소 연령 |
|---|---|---|
| KR | 네이버, 이메일 | 만 14세 |
| US | Google, Apple, X | 만 13세 |
| TW | Google, Apple, X | 만 18세 |
| JP | Google, Apple, X | 만 18세 |
| CN | WeChat, QQ | 만 18세 |

## 5. 구현된 핵심 기능 (현재까지)

- **플로우 A (IP 고정)**: 감지된 IP 국가를 그대로 적용, 사용자 변경 불가

### 9-2. 방향 논의 먼저
- **바로 코드부터 짜지 말 것**
- 작업 전 **선택지 제시 → 확인받기 → 진행**
- 유저는 고민/질문을 많이 하며 방향을 함께 결정하는 스타일

### 9-3. "여기까지 한 작업 저장" = 메모리 저장 요청

## 12. 회사 PC 이어받기 시 주의사항

### 12-1. 쓰기 권한 없음 (읽기 전용)
- 현재 회사 PC에는 GitHub 인증을 설정하지 않는 방향
- 회사 PC에서 `git push`는 **인증 오류**로 실패할 것
- **해결**: 회사에서 수정한 내용은 ZIP 또는 diff로 저장 → 집 PC에서 반영 후 push
- 또는 회사에서 수정한 파일 내용을 대화로 공유 → 집에서 수동 반영

### 12-2. Claude 메모리는 계정/PC마다 별개
- `C:\Users\*\.claude\projects\...`는 로컬 파일
- 회사 PC의 다른 Claude 계정은 집 PC 메모리를 볼 수 없음
- **대체**: 본 HANDOVER.md가 그 역할을 대신함
- 이어받는 쪽에서 새로 얻은 중요한 결정/피드백은 **HANDOVER.md에 추가 기록** 권장 → 집에서 머지

### 12-3. 수정은 로컬에서, 배포는 집에서
- 회사 PC에서 `git clone` → 수정 → 로컬 확인
- 변경 파일을 집 PC로 전달 (이메일, 본인 Drive, USB 등)
- 집 PC에서 적용 → commit → push → Vercel 자동 배포
