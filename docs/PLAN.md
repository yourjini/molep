# molep 글로벌 포털 — 미로그인 접근 + 회원가입 플로우 HTML 시안

## Context (배경)

- 글로벌 게임 포털 "molep" 신규 런칭 예정. 단일 도메인(예: `www.molep.com`)으로 운영하되, 접속 국가(IP)에 따라 회원가입 수단·약관·정책이 분기되어야 함.
- 차주 내부 회의에서 **비로그인 상태 접근 → 회원가입 완료**까지 시연 가능한 인터랙티브 HTML 프로토타입이 필요.
- 시연 목적이므로 실제 OAuth 연동은 하지 않지만, 국가 스위치로 플로우가 실시간 변경되고, 국가별 정책/리스크가 화면 옆에 **오버레이**로 표시되어야 이해관계자가 한 번에 파악 가능.
- 대상 국가: **한국(KR), 미국(US), 대만(TW), 일본(JP), 중국(CN)** 5개국.

## 핵심 설계 결정

### 1. "국가"와 "언어"는 분리 (중요)

| 항목 | 판단 기준 | 변경 가능 여부 | 영향 |
|---|---|---|---|
| **관할 국가(Jurisdiction)** | **서버 측 GeoIP** (MaxMind/ipinfo) | ❌ 고정 (VPN 추정 시 경고) | 가입 수단, 약관, 연령 기준, 미성년 동의, 실명제 |
| **표시 언어(UI Locale)** | IP 국가의 기본 언어 → 없으면 `Accept-Language` 헤더 → `en` 폴백 | ✅ 상단 언어 셀렉터로 자유 변경 | 화면 문구만 |

**근거**: IP는 법적 관할을 결정(약관·미성년 법규는 거주지 법률 적용)하지만, 언어는 사용자 편의. 한국 IP의 영어 사용자가 UI를 영어로 보되, 가입은 한국 약관/네이버 로그인을 따라야 함. **브라우저 언어를 Jurisdiction으로 쓰면 VPN 없이도 우회 가능해 규제 리스크 큼**.

### 2. 단일 도메인 + 엣지 분기

- `www.molep.com` 단일 도메인. CDN 엣지(Cloudflare Workers / AWS CloudFront Functions)에서 GeoIP 헤더 주입 → 응답 HTML은 동일하지만 `<html data-country="KR" data-locale="ko">` 로 힌트 제공.
- 초기 화면은 **정적 HTML + 클라이언트 하이드레이션**. 관리자 타겟팅 콘텐츠(비로그인용)는 엣지 KV 캐시로 국가별 버전을 프리-렌더.
- **중국은 별도 스택**(ICP 신청·版号 필요). 시연 범위에서는 CN 선택 시 "별도 운영 대상" 표시 + 시뮬레이션만.

### 3. 회원 DB 구조 (시연 범위 밖이지만 플로우 전제)

```
User {
  id, email_or_socialId,
  jurisdiction: "KR"|"US"|"TW"|"JP"|"CN",  // 가입 시점 IP 고정
  locale_pref: "ko"|"en"|"zh-TW"|"ja"|"zh-CN",
  auth_provider: "naver"|"email"|"google"|"apple"|"x"|"line"|"wechat",
  dob, is_minor, guardian_consent_at?, realname_verified_at?,
  terms_versions: { tos:"v2.1", privacy:"v1.4", marketing_opt:"v1.0", ... },
  consent_log: [...]  // 감사 로그(PIPA/GDPR/PIPL 요건)
}
```

## HTML 프로토타입 아키텍처

**단일 SPA**, 순수 HTML/CSS/Vanilla JS (빌드 없음 — 브라우저에서 `index.html` 열면 바로 실행). 이유: 시연·공유 편의, 팀원이 즉시 수정 가능.

### 디렉터리 구조 (작업 위치: `C:\autoqa\molep\auth\`)

> **경로 규칙**: 평소 작업 경로(`C:\autoqa\molep`, 전부 소문자)를 기준으로 통일. 앞으로 Claude Code 실행은 항상 `cd C:\autoqa\molep` → `claude`. 대소문자가 다른 경로(`C:\AutoQA\...`)로 진입하면 Claude Code가 별개 프로젝트로 인식해 세션/메모리가 분산되므로 금지.

### 화면 흐름 (해시 라우트)

```
#/landing               → 미로그인 홈 (상단: [국가:KR▼] [언어:한국어▼] | CTA: 로그인·회원가입)
                         → 히어로 + 관리자 타겟팅 배너 3건 (국가별) + 공통 콘텐츠
#/signup/method         → 가입 수단 선택 (국가에 따라 동적)
                         · KR: [네이버] [이메일]
                         · US: [Google] [Apple] [X]
                         · TW: [LINE] [Google] [Apple]
                         · JP: [LINE] [Google] [Apple] [X]
                         · CN: [WeChat] [QQ]  (상단 "시뮬레이션 안내" 배너: 실제 운영은 ICP·版号·국내 법인 필요)
#/signup/auth           → 선택한 인증 수단 시뮬레이션 (가짜 OAuth 페이지)
#/signup/terms          → 국가별 약관 동의
                         · 필수: ToS / Privacy
                         · 선택(국가별): 마케팅 / 위치기반(KR) / 크로스보더 이전(CN/KR)
                         · 약관 버전 표시 + "전체 동의" / 개별 토글
#/signup/age-gate       → 생년월일 입력 + 국가별 미성년 판정
                         · KR: <14 → guardian 분기
                         · US: <13 → COPPA VPC 분기 (또는 차단 옵션)
                         · TW: <18 → guardian 분기 (7-17 한정 행위능력)
                         · JP: <18 → guardian 분기 (결제 시점 주로)
                         · CN: <14 → guardian + <18 → anti-addiction 플래그
#/signup/guardian       → 법정대리인 동의 (국가별 방식)
                         · KR: KMC 본인인증 샘플 UI
                         · US: 신용카드 $0 / 신분증 업로드 선택지
                         · TW/JP: 이메일 + 카드 확인
                         · CN: 실명 ID + guardian 실명
#/signup/realname       → 실명제 (CN 필수, KR 결제 연동 시)
                         · CN: 身份证(18자리) 형식 검증 + 이름 한자 입력 + 얼굴인식 자리표시자
                         · KR: PASS/KMC 본인인증 가짜 팝업
#/signup/cn-antiaddict  → (CN 전용) 미성년 방침몰시스템 안내
                         · "<18세 주 3시간 제한 (금·토·일 20~21시)" 배너
                         · 크로스보더 데이터 전송 별도 동의 (PIPL)
                         · 데이터 로컬라이즈 안내
#/signup/welcome        → 완료 + 가입 summary (jurisdiction, consent log 표시)
                         · CN의 경우 추가로 "이 세션은 시뮬레이션입니다" 고지
```

### 국가 스위처 동작

- 우상단에 **[🌐 Demo: Force Country]** 드롭다운 (데모 전용, 프로덕션에서는 숨김).
- 변경 시 `state.country` 업데이트 → 현재 화면 재렌더 + URL에 `?geo=KR` 쿼리 유지 (공유 가능).
- **언어 셀렉터**는 별도: 어떤 국가든 ko/en/zh-TW/ja/zh-CN 5개 자유 전환. 다만 약관 원문은 관할 국가 기준으로 고정(번역본만 제공).

### 리스크 오버레이 패널

- **디자인은 사용자가 Figma 파일 제공 예정** → 수령 후 스타일/레이아웃 확정.
- 기능 요건(디자인과 무관하게 확정):
  - 현재 스텝 × 현재 국가 조합에 해당하는 **리스크 카드** 렌더
  - 심각도 배지 (High/Med/Low)
  - 리스크 설명 (법적 / UX / 기술 / 성능)
  - "이 시점에서 이렇게 동작해야" 가이드
  - 관련 법규 링크(샘플)
- 데이터 소스: `risks.json`, `docs/risks-register.md`와 동기화.
- **Figma 수령 전에는** `risk-panel.css`에 최소 뼈대만 두고 컴포넌트(HTML 구조·데이터 바인딩)만 먼저 완성해둠. 디자인 수령 즉시 스타일 교체.

### 국가별 핵심 리스크 (플로우 포인트 기준)

**🇰🇷 KR**
- `age-gate`: 만 14세 미만 식별 시 법정대리인 동의 없이는 가입 진행 불가 (PIPA §22-2). — *동의 안 받으면 과태료·행정처분.*
- `terms`: "개인정보 제3자 제공" 동의를 "서비스 이용약관"과 분리 제공 필수. — *묶음 동의 금지.*
- `welcome` 이후: 게임시간 선택제 안내 (미성년 가입 시) — 가이드 배너 & 보호자 설정 메뉴 노출.
- 확률형 아이템이 있다면 가입 전 노출 콘텐츠에도 공시 문구(2024.3.22~).

**🇺🇸 US**
- `age-gate`: **Neutral age screen** — 실패 시 다시 돌아가 변경 시도 금지 (FTC 가이드). 세션 쿠키로 재시도 차단.
- `terms`: CCPA "Do Not Sell or Share" 링크 상시 노출, GPC 헤더 존중.
- CA 접속자: AADC 고-프라이버시 기본값 (정밀 위치·행동 타겟팅 OFF).
- COPPA <13: VPC 완료 전 어떠한 개인정보도 수집 금지 → 가입 막히면 **즉시 폐기**.

**🇹🇼 TW**
- `age-gate`: 7~17세는 제한적 행위능력 → 법정대리인 동의 없이 유료 거래 불가. 무료 가입도 보호자 동의 UX 권장.
- `terms`: PDPA 민감정보(의료·범죄 등) 수집하지 않음을 명시. 대중국 크로스보더 제한 고려 — 호스팅 리전 주의.
- 게임 등급 뱃지 노출 의무 (0/6/12/15/18).

**🇯🇵 JP**
- `age-gate`: 18세 미만 결제 시 보호자 확인 플로우. 가챠가 있다면 월 상한(CESA 가이드라인: ~10만엔).
- `terms`: 전기통신사업법(2023) — 외부송신(쿠키·SDK) 고지 별도 화면 또는 상설 링크.
- 가가와현 IP 접속 시 플레이 시간 권고 안내 (선택적).

**🇨🇳 CN**
- `method`: 시연 중 "CN은 별도 운영/법인 필요 (ICP·版号)" 경고 배너. 실제 서비스 전 법무 검토 필수.
- `age-gate`: 실명 + 얼굴인식 스팟체크(국가 방침몰인증평대). <18는 금·토·일 20~21시만 1시간 플레이.
- `terms`: PIPL — 크로스보더 전송에 별도 동의 + CAC 심사/SCC/인증. 민감정보·미성년은 별도 동의.
- 호스팅: 중국 본토 데이터는 로컬라이즈. 해외 스택에서 서빙하면 방화벽·속도·규제 리스크.

## 반응형 설계 전략

- **모바일 우선(Mobile-first) CSS** — 기본 스타일은 모바일, `@media (min-width: 768px)` / `(min-width: 1024px)`로 확장.
- **브레이크포인트**: `sm=480 / md=768 / lg=1024 / xl=1280`.
- **레이아웃 변형**:
  - 국가·언어 스위처: 데스크톱은 우상단 드롭다운, 모바일은 햄버거 메뉴 내부로 이동.
  - 가입 수단 버튼: 데스크톱 가로 그리드, 모바일 세로 스택 (44px 이상 터치 타겟).
  - 리스크 오버레이 패널: 데스크톱 우측 고정, 모바일은 화면 하단 "리스크 보기" 시트(Bottom Sheet)로 슬라이드업.
  - 약관 화면: 데스크톱 2단(약관 본문 + 동의 토글), 모바일 1단 스크롤.
- **입력 필드**: `type="date"`, `type="tel"`, `inputmode` 속성으로 모바일 키보드 최적화.
- **이미지**: `srcset` + `sizes`로 DPR/뷰포트별 적정 해상도. 배너는 모바일용 세로 크롭 이미지 별도 제공.
- 시연 시 크롬 DevTools 디바이스 에뮬레이션(iPhone 15, Galaxy S23, iPad) 3종으로 실시간 전환 데모.

## 성능 고려사항 (비로그인 진입 속도)

1. **엣지 SSR로 비로그인 타겟팅 배너 프리렌더**. 국가별 5 버전을 Cloudflare KV/CloudFront Functions 캐시. TTL 5분, 관리자 발행 시 퍼지.
2. **Admin 콘텐츠 API 분리**: 로그인 불필요 구간은 읽기 전용 CDN 캐시. DB 조회는 관리자 저장 시 정적 JSON 생성(ISR 방식).
3. **핵심 웹 바이탈 목표**: LCP < 2.5s, INP < 200ms, CLS < 0.1. 국가별 지연 계산(KR→서울 리전, JP→도쿄, TW→싱가포르 or 대만, US→us-east, CN→별도).
4. **프로토타입 단계에선 순수 정적 파일** — 성능 리스크 카드를 리스크 패널에 별도 수록(시연 시 설명).
5. **이미지**: 국가별 배너는 AVIF + WebP 폴백, `fetchpriority=high`는 히어로 1장만.
