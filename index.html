🔀 대안 플로우 B: 수동 국가 변경 허용 (에픽게임즈 방식)

IP 자동 감지 + 수동 변경 옵션을 제공하는 방식. 현재 플로우 A(IP 고정)와 비교용.

플로우 A: IP 고정 (현재 채택)

채택
사이트 접속
↓
IP 감지 → 국가 확정 (변경 불가)
↓
해당 국가 언어/인증/약관 표시
↓
회원가입 진행

✅ 장점

  • 법적 관할권 자동 매칭 (안전)
  • 어뷰징 원천 차단
  • 구현 단순
  • 약관 혼선 없음

❌ 단점

  • VPN/해외교민 UX 저하
  • IP 오탐 시 대응 없음
  • 출장자가 원래 국가 서비스 이용 불가

플로우 B: 수동 변경 허용 (에픽 방식)

선택지
사이트 접속
↓
IP 감지 → 국가 자동 선택
↓
"🇰🇷 한국이 맞나요?"
[맞습니다] [다른 국가 선택 ▾]
↓
맞습니다 → 그대로 진행
다른 국가 → 국가 선택 후 진행
⚠️ 가입 후 변경 불가 고지
↓
선택한 국가 기준 인증/약관 표시
↓
회원가입 완료 (국가 고정)

✅ 장점

  • VPN/교민/출장자 대응 가능
  • IP 오탐 시 사용자가 직접 수정
  • 에픽게임즈 등 글로벌 선례 있음
  • UX 유연성 높음

❌ 단점

  • 어뷰징 가능 (가격 차별화 우회)
  • 잘못된 국가 선택 시 약관 불일치 법적 리스크
  • 미성년자 규제 우회 가능성 (KR→US 선택 시 14세→13세)
  • CS 부담 증가
⚠️ 플로우 B 채택 시 필수 안전장치
  • 가입 후 국가 변경 불가 (또는 1회 제한, CS 통해서만)
  • 국가 변경 시 "선택한 국가의 법률이 적용됩니다" 고지 + 재동의
  • CN 선택 시 반드시 실명인증 강제 (우회 불가)
  • 가격 어뷰징 방지: 결제 시 현재 IP 국가와 가입 국가 불일치 감지 → 제한
  • 변경 이력 감사 로그 남기기

💡 권장안: 플로우 A(IP 고정)로 런칭 후, 교민/출장자 CS 문의 비율 모니터링. 일정 비율 초과 시 플로우 B로 전환 검토. 또는 하이브리드: IP 고정이되, 가입 전 "국가가 맞지 않나요?" 링크를 두어 클릭 시 CS/고객센터로 연결 (수동 변경은 CS에서만 처리).
