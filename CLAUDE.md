# molep 글로벌 포털 — 프로젝트 메모리

> 이 파일은 Claude Code가 세션 시작 시 자동으로 읽는 프로젝트 컨텍스트입니다.
> 여기에 적힌 규칙과 참조는 모든 대화에 기본 전제로 적용됩니다.

## 프로젝트 개요

- **제품명**: molep (글로벌 게임 포털)
- **형태**: 단일 도메인(`www.molep.com`) + GeoIP 기반 국가 분기
- **대상 국가**: 🇰🇷 KR · 🇺🇸 US · 🇹🇼 TW · 🇯🇵 JP · 🇨🇳 CN (5개국)
- **현 단계**: 비로그인 접근 + 회원가입 플로우 **인터랙티브 HTML 프로토타입** (차주 시연용)

## 핵심 참고 문서 (반드시 먼저 읽을 것)

설계 결정·구현 범위·리스크는 아래 플랜이 단일 진실 원천(Single Source of Truth):

@docs/PLAN.md

플랜과 충돌하는 요청이 들어오면 먼저 플랜과의 차이를 지적하고 사용자에게 확인받을 것.

## 작업 경로 규칙 (중요)

- **캐노니컬 경로**: `C:\autoqa\molep` (**전부 소문자 고정**)
- Claude Code 실행은 항상 `cd C:\autoqa\molep` 후 `claude`
- **금지**: `C:\AutoQA\...` 등 대소문자가 다른 경로로 진입 — Claude Code가 별개 프로젝트로 인식해 세션/메모리가 분산됨
- HTML 프로토타입 산출물은 모두 `C:\autoqa\molep\auth\` 하위에 생성

## 기술 스택 & 코드 스타일

- **순수 HTML + CSS + Vanilla JS** (빌드 도구·프레임워크 없음)
- 모바일 우선 반응형: 브레이크포인트 `sm=480 / md=768 / lg=1024 / xl=1280`
- 터치 타겟 최소 44px
- 한중일 글자 깨짐 방지 위해 시스템 폰트 스택 사용 (`-apple-system`, `"Noto Sans KR"`, `"Noto Sans JP"`, `"Noto Sans TC"`, `"Noto Sans SC"`)
- 해시 기반 라우터 (`#/landing`, `#/signup/method` 등)
- 상태 관리: 단일 `state.js` 저장소
